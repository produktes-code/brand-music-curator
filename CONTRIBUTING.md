# Contributing to Brand Music Curator

Thank you for your interest in contributing to **Brand Music Curator**! To maintain a professional and stable codebase, please follow these contribution guidelines.

## 🐛 Reportando Bugs (Reporting Bugs)
*   **Canal:** Utiliza la pestaña de **Issues** en GitHub.
*   **Requisitos:** 
    *   Describe de forma clara el comportamiento observado y el comportamiento esperado.
    *   Incluye los pasos precisos para reproducir el error.
    *   Adjunta capturas de pantalla, videos del error, y los logs del backend ubicados en `logs/combined.log`.
    *   Especifica tu sistema operativo y la versión de Node.js/Python.

## 💡 Sugiriendo Features (Suggesting Features)
*   **Canal:** Crea un **Issue** en GitHub con la etiqueta (label) `enhancement`.
*   **Requisitos:**
    *   Explica detalladamente el caso de uso y el beneficio comercial/técnico.
    *   Propón una solución o flujo de interfaz si es posible.

## 🔀 Pull Requests (PR)
1.  Haz un **Fork** del repositorio.
2.  Crea una rama descriptiva para tu cambio utilizando prefijos semánticos:
    *   `feature/nueva-funcionalidad`
    *   `fix/correccion-de-bug`
3.  Asegúrate de respetar el estilo de código existente:
    *   Usa ESLint/Prettier para dar formato al frontend.
    *   Asegúrate de que la aplicación compila ejecutando `npm run build` en el frontend.
    *   No dejes sentencias `console.log` o `print` de depuración sueltas; usa los módulos de logging oficiales.
4.  Abre el Pull Request apuntando a la rama `main` del repositorio original y describe tus cambios en la plantilla.
